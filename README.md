# CodeCoach AI 🧠

> **An AI coding tutor that uses knowledge graphs to find WHY you're stuck, not just WHAT you got wrong**

---

## 🎯 The Problem

Traditional learning platforms tell you **WHAT** you got wrong, but not **WHY**:

- ❌ **LeetCode**: "Wrong Answer"
- ❌ **CodeChef**: "Try a harder problem"
- ❌ **Coursera**: "Watch this 2-hour video"

**The Real Issue**: Students fail recursion not because they don't understand recursion, but because they're missing the prerequisite: **stack frames**.

---

## 💡 Our Solution

**CodeCoach AI** treats learning as a **graph, not a list**.

| Traditional Tutors | CodeCoach AI |
|-------------------|--------------|
| "You got recursion wrong." | "You got recursion wrong because you're missing **stack frames**. Study stack frames first, then recursion will click." |

When a student fails a problem, CodeCoach:
1. ✅ Traverses **4 hops** through their prerequisite graph
2. ✅ Calculates their **knowledge debt score**
3. ✅ Predicts what they'll **fail next week**
4. ✅ Matches them with **peers** who overcame the same gap
5. ✅ Asks **Socratic questions** targeting the exact missing concept

**No SQL database can do this. Only a graph.**

## 🚀 Key Features

### 🧠 Graph-Powered Skill Map
Interactive knowledge graph showing:
- **Red nodes**: Weak concepts
- **Orange nodes**: Missing prerequisites
- **Green nodes**: Mastered concepts
- **Arrows**: Prerequisite dependencies

Powered by TigerGraph's `skillIntelligence` GSQL query with multi-hop traversal.

### 🤖 Graph AI Chat (LangChain ReAct Agent)
AI mentor that autonomously queries your knowledge graph:
- **You ask**: "Why am I struggling with trees?"
- **Agent thinks**: "I need their skill gaps"
- **Agent calls**: `get_skill_gaps` tool → TigerGraph
- **Agent observes**: "Missing prerequisite: recursion"
- **Agent answers**: "You're struggling with trees because you haven't mastered recursion yet."

Badge system shows **"Graph-Powered"** vs **"General AI"** responses.

### ⚔️ Arena Battle Mode
1v1 coding contests with AI-generated problems:
- Matchmaking uses `arenaMatchmaking` GSQL query (4-hop traversal)
- Finds opponents with **overlapping weaknesses**
- AI generates problems targeting **both players' weak concepts**
- Code executed via **Piston API** (sandboxed)
- Results update TigerGraph edges in **real-time**

**Modes**: Student vs Student, Student vs AI (adaptive difficulty)

### 📊 Knowledge Debt Dashboard
Quantifies accumulated conceptual gaps:
```
debt_score = error_frequency × blocks_count
total_debt = sum(all debt_scores)
```

**Debt Levels**:
- 🔴 **Critical** (>200): Urgent intervention needed
- 🟠 **High** (>100): Focus on fundamentals
- 🟢 **Moderate** (≤100): On track

### 🔮 GNN-Lite Prediction
Predicts which concepts you'll struggle with **before** attempting them:
- Inspired by Graph Neural Networks (Kipf & Welling, 2017)
- Weakness signals propagate through prerequisite edges
- Weighted by **PageRank centrality** (concept importance)
- Example: *"You will likely score 34% on Graph Traversal next week"*

### 💬 Socratic Graph Questioning
Instead of giving answers, asks targeted questions:
- **Traditional**: "Recursion uses a base case and recursive call."
- **CodeCoach**: "What happens to the call stack when a function calls itself?"

Questions generated by traversing your knowledge graph to find the weakest prerequisite.

### 🎯 Impostor Syndrome Detection
Identifies students with **high quiz scores but weak fundamentals**:
- Powered by `findImpostors` GSQL query
- Filter: `avg_quiz_score > 75 AND many weak_in edges`
- Early intervention prevents future struggles

## 🗄️ TigerGraph Integration

### Graph Schema
**Vertices**: `Student`, `Concept`  
**Edges**: `weak_in` (error_frequency), `prerequisite` (importance)

### 5 Custom GSQL Queries

| Query | Purpose | Traversal |
|-------|---------|-----------|
| `skillIntelligence` | Root cause analysis | Student → weak_in → Concept → reverse_prerequisite |
| `knowledgeDebt` | Debt scoring | Student → weak_in → Concept (sorted) |
| `arenaMatchmaking` | Peer matching | Student → weak_in → Concept ← weak_in ← Student (4-hop) |
| `findImpostors` | High scorers with gaps | Student → weak_in → Concept (filter) |
| `tg_pagerank` | Concept importance | Built-in GDSL algorithm |

All queries run in **real-time** via TigerGraph REST API.

### Why TigerGraph?
✅ **Prerequisite graph traversal** (not just skill tags)  
✅ **Real-time edge updates** (graph learns as you use it)  
✅ **Multi-hop reasoning** (4-hop matchmaking, not 1-level lookup)  
✅ **GDS library** (tg_pagerank for concept importance)

1. **Sign in** with Google → User created in DynamoDB

## 🔬 Technical Highlights

### LangChain ReAct Agent
- Based on **Yao et al. 2023 ReAct paper**
- Autonomous tool selection with transparent reasoning
- Shows intermediate steps for explainability

### GNN-Lite Prediction
- Inspired by **Kipf & Welling 2017 GCN paper**
- Message passing through graph edges
- PageRank-weighted propagation

### Real-time Graph Updates
- Every quiz/arena result → TigerGraph edge upsert
- Conditional writes prevent race conditions
- Fire-and-forget for non-blocking UX
